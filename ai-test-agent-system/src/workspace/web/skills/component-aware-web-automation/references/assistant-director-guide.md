# 副导演指南 (Agent 6: Assistant Director)

## 使命与哲学

副导演是**翻译者**——将编舞师的抽象旅程计划映射为具体的确定性 Playwright 测试代码，**仅使用 POM 方法**。

> **核心纪律**: 测试代码中绝不使用原始定位器。绝不使用 `page.locator()` 或 `page.$()`。绝不使用 CSS 选择器或 XPath。绝不硬编码等待时间。所有交互通过 POM 语义动作方法完成。

这是确定性生成的关键：**同源代码 → 同测试代码**。不依赖运行时发现，不依赖人为判断。

---

## 测试生成流程

```
输入:
  journeys.json          (编舞师规划的旅程)
  poms/*.ts              (布景师生成的POM类)
  locator-catalog.json   (调度教练的定位器目录)
  permission-model.json  (权限模型)

处理:
  1. 为每个旅程确定所需的 POM 实例
  2. 映射每个步骤到 POM 方法调用
  3. 生成 beforeEach 状态排列设置代码
  4. 生成测试体（纯 POM 方法调用序列）
  5. 添加预期结果断言

输出:
  tests/*.spec.ts        (确定性 Playwright 测试文件)
```

## 确定性规则（6条铁律）

### 规则 1: 同源代码 → 同测试代码

**定义**: 给定相同的输入文件（component-registry + journeys + poms），生成过程必须产生完全相同的测试代码。没有随机性，没有运行时发现。

**实践**: 使用确定性排序（按组件名、旅程ID排序），不依赖文件系统遍历顺序。

### 规则 2: 测试代码中绝不使用 page.locator() 或 page.$()

**定义**: 所有 DOM 交互通过 POM 方法完成。测试文件中不应出现任何 `page.locator()`、`page.$()`、`page.$$()` 调用。

**原因**: POM 是定位器的封装边界。测试只需表达业务意图，不需知道定位器细节。

### 规则 3: 绝不使用 CSS 选择器或 XPath

**定义**: 测试代码中不应出现任何字符串形式的 CSS 选择器或 XPath。

**扩展**: 即使在 POM 内部，也应优先使用 `page.getByTestId()`、`page.getByRole()` 等语义定位器。

### 规则 4: 绝不硬编码等待时间

**定义**: 不使用 `page.waitForTimeout(5000)` 或类似的硬编码等待。所有等待通过 POM 的 `waitForReady()` 或具体的状态断言完成。

**原因**: 硬编码等待是 flaky test 的根源。要么元素在合理时间内出现（断言通过），要么不出现（断言失败）。

### 规则 5: 每个测试覆盖恰好一个排列

**定义**: 一个 `test()` 块对应一个旅程，一个旅程对应一个排列。测试体内不应有 `if/else` 分支逻辑。

**原因**: 条件分支意味着一个测试在"运行时决定"要验证什么，违反确定性原则。

**例外**: 测试清理代码（afterEach）可以有条件逻辑。

### 规则 6: 测试命名遵循排列格式

**定义**: 测试命名格式为 `{Component}-{Role}-{FlagState}-{Action}`

**示例**:
- `UserTable-admin-showDeleteOn-deleteRow`
- `LoginForm-any-correctPassword-loginSuccess`
- `Dashboard-editor-defaultSidebar-viewLimitedMenu`

**原因**: 从测试名即可知道它覆盖的是哪个排列，无需查看代码。

## 排列设置模式

### Feature Flag 设置

```typescript
test.beforeEach(async ({ page }) => {
  // 通过 page.evaluate() 在应用初始化前设置 Feature Flag
  await page.evaluate(() => {
    // React: 设置全局配置
    window.__FEATURE_FLAGS__ = {
      showDeleteButton: true,
      enableNewDashboard: false
    };

    // Vue: 设置 Vuex/Pinia store
    // window.__VUE_STORE__?.commit('SET_FLAG', { key: 'showDelete', value: true });

    // Angular: 设置 Service
    // window.__ANGULAR_FLAGS__ = { ... };
  });

  loginPage = new LoginPage(page);
  await loginPage.navigateTo();
});
```

### 角色设置（登录模拟）

```typescript
test.beforeEach(async ({ page }) => {
  // 方式1: 通过登录流程设置角色
  loginPage = new LoginPage(page);
  await loginPage.navigateTo();
  await loginPage.loginAsRole('admin');

  // 方式2: 通过注入认证状态（更快速、更稳定）
  await page.evaluate((token) => {
    localStorage.setItem('auth_token', token);
    localStorage.setItem('user_role', 'admin');
  }, 'admin-mock-token');

  dashboardPage = new DashboardPage(page);
  await dashboardPage.navigateTo();
});
```

### 数据条件设置

```typescript
test.beforeEach(async ({ page }) => {
  // 方式1: 通过 API mock 设置数据条件
  await page.route('**/api/users', async (route) => {
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify([])  // 空数据
    });
  });

  // 方式2: 错误数据条件
  await page.route('**/api/users', async (route) => {
    await route.fulfill({ status: 500, body: 'Server Error' });
  });
});
```

### 组合排列设置

```typescript
test.describe('UserTable - admin - showDelete on - populated', () => {
  let userTablePage: UserTablePage;

  test.beforeEach(async ({ page }) => {
    // 设置 Feature Flag
    await page.evaluate(() => {
      window.__FEATURE_FLAGS__ = { showDeleteButton: true };
    });

    // 设置角色（注入认证）
    await page.evaluate(() => {
      localStorage.setItem('auth_token', 'admin-token');
      localStorage.setItem('user_role', 'admin');
    });

    // 设置数据条件（mock API返回有数据）
    await page.route('**/api/users', async (route) => {
      await route.fulfill({
        status: 200,
        contentType: 'application/json',
        body: JSON.stringify([
          { id: 1, name: 'Alice', email: 'alice@example.com' },
          { id: 2, name: 'Bob', email: 'bob@example.com' },
          { id: 3, name: 'Charlie', email: 'charlie@example.com' }
        ])
      });
    });

    userTablePage = new UserTablePage(page);
    await userTablePage.navigateTo();
    await userTablePage.waitForReady();
  });

  test('admin can delete a row when delete button visible', async () => {
    await userTablePage.assertDeleteButtonVisible();
    await userTablePage.deleteRow(3);
    await userTablePage.assertRowDeleted(3);
    await userTablePage.assertRowCount(2);
  });
});
```

## 关键原则

1. **翻译而非创作** — 副导演将旅程映射为代码，不添加新逻辑
2. **POM 是唯一接口** — 测试代码只看得到 POM 方法，看不到定位器
3. **确定性优先** — 同输入同输出，可审计、可复现
4. **排列即测试** — 一个排列一个测试，命名即描述
5. **状态设置在 beforeEach** — 排列参数在测试执行前已完全确定
